# 勉強くん 画面遷移設計

## 1. このファイルの目的

このドキュメントは、iPhoneアプリ「勉強くん」の画面遷移を定義する。

初期版では、下部タブを中心としたシンプルな遷移にする。

---

## 2. 基本ナビゲーション方針

## 2-1. 下部タブ

メイン画面は下部タブで切り替える。

| タブ | 画面 |
|---|---|
| ホーム | `HomeView` |
| 問題演習 | `QuizSettingView` |
| 復習 | `ReviewView` |
| 記録 | `StudyRecordView` |
| 設定 | `SettingsView` |

## 2-2. NavigationStack

各タブごとに `NavigationStack` を持たせる。

```swift
TabView {
    NavigationStack { HomeView() }
    NavigationStack { QuizSettingView() }
    NavigationStack { ReviewView() }
    NavigationStack { StudyRecordView() }
    NavigationStack { SettingsView() }
}
```

## 2-3. 下部タブを非表示にしてよい画面

以下は学習に集中する画面なので、下部タブを非表示にしてよい。

- `QuizView`
- `AnswerResultView`
- `QuizCompleteView`

---

# 3. AppRoute設計

## 3-1. 目的

SwiftUI の `navigationDestination` で使う遷移先を定義する。

## 3-2. 定義案

```swift
enum AppRoute: Hashable {
    case quizSetting
    case quiz
    case answerResult
    case quizComplete
    case review
    case questionCreate
    case questionEdit(questionId: UUID)
    case questionList
    case studyRecord
    case goalSetting
    case settings
}
```

ただし、`QuizView` / `AnswerResultView` / `QuizCompleteView` は `QuizSessionViewModel` の状態を共有するため、単純な route だけでなく、親Viewで状態を保持して画面を切り替える実装でもよい。

補足：`answerResult` は実装上、正誤・回答・問題情報を表示する必要がある。`AppRoute` だけで扱いにくい場合は、`QuizSessionViewModel` に現在の回答結果を保持し、`QuizSessionView` の内部状態で `QuizView` / `AnswerResultView` / `QuizCompleteView` を切り替えてよい。

---

# 4. メイン導線

## 4-1. ホームから問題を解く

```text
HomeView
  → QuizSettingView
  → QuizView
  → AnswerResultView
  → QuizView
  → AnswerResultView
  → QuizCompleteView
```

## 4-2. 処理詳細

| 画面 | 処理 |
|---|---|
| `HomeView` | 「問題を解く」を押す |
| `QuizSettingView` | 出題方法・分野・問題数を選択 |
| `QuizView` | 選択肢をタップして回答 |
| `AnswerResultView` | 正解・不正解・解説を確認 |
| `QuizCompleteView` | 今回の結果を確認 |

## 4-3. 遷移条件

| 操作 | 条件 | 遷移先 |
|---|---|---|
| この条件で開始 | 問題あり | `QuizView` |
| この条件で開始 | 問題なし | エラー表示 |
| 選択肢タップ | 常に | `AnswerResultView` |
| 次の問題へ | 次の問題あり | `QuizView` |
| 次の問題へ | 最後の問題 | `QuizCompleteView` |

---

# 5. 復習導線

## 5-1. ホームから復習する

```text
HomeView
  → ReviewView
  → QuizView
  → AnswerResultView
  → QuizCompleteView
```

## 5-2. 下部タブから復習する

```text
MainTabView
  → ReviewView
  → QuizView
  → AnswerResultView
  → QuizCompleteView
```

## 5-3. 遷移条件

| 操作 | 条件 | 遷移先 |
|---|---|---|
| 復習を開始 | 復習対象あり | `QuizView` |
| 復習を開始 | 復習対象なし | 空データ表示 |
| 復習で正解 | 2回連続正解なら復習対象解除 | `AnswerResultView` |
| 復習で不正解 | 復習対象を継続 | `AnswerResultView` |

---

# 6. 問題管理導線

## 6-1. 設定から問題一覧へ

```text
SettingsView
  → QuestionListView
```

## 6-2. 問題一覧から問題追加へ

```text
QuestionListView
  → QuestionCreateView
```

## 6-3. 問題一覧から問題編集へ

```text
QuestionListView
  → QuestionEditView
```

## 6-4. ホームから問題追加へ

```text
HomeView
  → QuestionCreateView
```

## 6-5. 問題追加完了後

```text
QuestionCreateView
  → QuestionListView または 前画面へ戻る
```

推奨：

- 問題一覧から追加した場合：問題一覧へ戻る
- ホームから追加した場合：ホームへ戻る、または保存完了後に閉じる

---

# 7. 学習記録導線

## 7-1. ホームから学習記録へ

```text
HomeView
  → StudyRecordView
```

## 7-2. 下部タブから学習記録へ

```text
MainTabView
  → StudyRecordView
```

## 7-3. 学習記録から分野別演習へ

分野別成績をタップした場合、将来的にはその分野の問題演習へ遷移してもよい。

初期版では必須ではない。

```text
StudyRecordView
  → QuizSettingView(selectedCategory: tappedCategory)
```

---

# 8. 目標設定導線

## 8-1. ホームから目標設定へ

```text
HomeView
  → GoalSettingView
```

## 8-2. 設定から目標設定へ

```text
SettingsView
  → GoalSettingView
```

## 8-3. 保存後

```text
GoalSettingView
  → 前画面へ戻る
```

保存後にホーム画面へ反映する。

---

# 9. 設定導線

## 9-1. 下部タブから設定へ

```text
MainTabView
  → SettingsView
```

## 9-2. 設定画面内の遷移

| メニュー | 遷移先・処理 |
|---|---|
| 目標設定 | `GoalSettingView` |
| 問題一覧 | `QuestionListView` |
| データリセット | 確認ダイアログ |
| アプリ情報 | アプリ情報表示 |

## 9-3. ハンバーガーメニュー

画面デザイン上にハンバーガーメニューがある場合でも、初期版では削除推奨。

残す場合は、タップ時に簡易メニューを表示する。

---

# 10. 回答結果画面の遷移仕様

## 10-1. 正解時

```text
QuizView
  → AnswerResultView(isCorrect: true)
```

表示：

- 正解！
- いいペースです！
- あなたの回答
- 正解
- 解説
- 次の問題へ

## 10-2. 不正解時

```text
QuizView
  → AnswerResultView(isCorrect: false)
```

表示：

- 不正解
- 復習して覚えましょう
- あなたの回答
- 正解
- 解説
- 復習リストに追加されました
- 次の問題へ

---

# 11. 演習終了の遷移仕様

## 11-1. 演習中の終了ボタン

```text
QuizView
  → 終了確認ダイアログ
```

確認文言：

```text
演習を終了しますか？
```

選択肢：

- 続ける
- 終了する

## 11-2. 終了する場合

```text
QuizView
  → QuizCompleteView
```

未回答の問題は結果に含めない。

---

# 12. データリセットの遷移仕様

```text
SettingsView
  → データリセットをタップ
  → 確認ダイアログ
  → 削除する
  → データ削除
  → SettingsViewに戻る
```

削除後、ホーム・記録・復習画面の表示は空状態になる。

---

# 13. 戻る操作

## 13-1. 通常画面

通常の詳細画面は左上の戻るボタン、または iOS 標準の戻るで前画面に戻る。

## 13-2. 問題演習中

問題演習中は誤操作防止のため、戻る場合も確認を出す。

```text
演習を終了しますか？
```

## 13-3. 入力フォーム中

問題追加・編集で未保存の入力がある場合、キャンセル時に確認を出す。

```text
入力内容を破棄しますか？
```

# 15. 実装上の注意

1. メインタブは5つ固定。
2. `AnswerResultView` は正解・不正解で分けない。
3. 問題演習中は下部タブ非表示でよい。
4. フォーム未保存時のキャンセルは確認を挟む。
5. データリセットは必ず確認ダイアログを挟む。
6. 条件に合う問題がない場合は画面遷移せずエラー表示する。
7. 復習対象がない場合は演習画面へ進めず空状態を表示する。
