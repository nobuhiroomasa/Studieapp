# 勉強くん 実装順序

## 1. このファイルの目的

このドキュメントは、iPhoneアプリ「勉強くん」を SwiftUI / SwiftData で実装する順番を定義する。

Codex に作業させる場合、一度に全画面を実装させると崩れやすいため、段階的に進める。

---

# 2. 実装全体方針

## 2-1. 最優先

まずは「問題を解けること」を最優先にする。

最初に完成させる体験：

```text
問題演習設定
  → 問題演習
  → 回答結果
  → 演習完了
```

## 2-2. 後から作るもの

以下は、問題演習の基本動作ができてから追加する。

- 復習機能
- 問題追加・編集
- 学習記録
- 目標設定
- ホーム画面の集計表示
- 設定画面

---

# 7. Phase 4：問題演習フロー実装

## 目的

最重要機能である4択問題演習を完成させる。

## 対象画面

- `QuizSettingView`
- `QuizView`
- `AnswerResultView`
- `QuizCompleteView`

## 作業内容

1. `QuizSettingView` で条件選択を実装
2. `QuizService` で条件に合う問題を取得
3. `QuizSessionViewModel` で演習状態を管理
4. `QuizView` で問題文・選択肢を表示
5. 選択肢タップで回答確定
6. `AnswerHistory` を保存
7. 正解・不正解を判定
8. `AnswerResultView` で結果表示
9. 最後の問題後に `QuizCompleteView` を表示

## 完了条件

- ランダム出題で問題を解ける
- 4択をタップすると正解・不正解が表示される
- 解説が表示される
- 最後に演習結果が表示される
- 回答履歴が保存される

---

# 8. Phase 5：復習機能実装

## 目的

間違えた問題を復習できるようにする。

## 対象画面

- `ReviewView`
- `QuizView`
- `AnswerResultView`

## 作業内容

1. 不正解時に `ReviewStatus` を作成・更新
2. `ReviewView` で復習対象を一覧表示
3. 復習開始で復習対象のみ出題
4. 復習で正解した場合、連続正解数を更新
5. 復習で2回連続正解したら復習対象から外す
6. 復習対象がない場合は空状態を表示

## 完了条件

- 不正解問題が復習画面に表示される
- 復習を開始できる
- 復習で2回連続正解した問題が対象外になる
- 復習対象なしの場合に空表示が出る

---

# 16. Codexへの依頼順テンプレート

## 16-1. 1回目

```text
SwiftUI / SwiftData のiPhoneアプリ「勉強くん」を作成します。
まずはプロジェクトの基本構成、フォルダ構成、Docs/MD・Docs/ScreenDesigns・Docs/ScreenSpecsの配置、MainTabView、空の12画面View、共通カラー定義を実装してください。
まだ各画面の詳細機能は実装しなくてよいです。
```

## 16-2. 2回目

```text
SwiftDataモデルを実装してください。
Question、AnswerHistory、ReviewStatus、StudyGoal、AppSetting と、関連enumを作成してください。
問題は4択固定、削除はisDeletedによる論理削除にしてください。
```

## 16-3. 3回目

```text
SampleQuestions.json と SampleDataSeeder を作成し、初回起動時だけ第二種電気工事士のサンプル問題を投入する処理を実装してください。
```

## 16-4. 4回目

```text
問題演習フローを実装してください。
QuizSettingView、QuizView、AnswerResultView、QuizCompleteViewを作成し、4択選択、正誤判定、回答履歴保存、解説表示、演習完了表示まで実装してください。
```

## 16-5. 5回目

```text
復習機能を実装してください。
不正解時にReviewStatusを更新し、ReviewViewに復習対象を表示してください。
復習で2回連続正解したら復習対象から外してください。
```

## 16-6. 6回目

```text
問題追加・一覧・編集機能を実装してください。
QuestionCreateView、QuestionListView、QuestionEditView、QuestionFormViewを作成し、バリデーション、保存、更新、論理削除まで実装してください。
```

## 16-7. 7回目

```text
学習記録、目標設定、ホーム画面、設定画面を実装してください。
回答履歴から累計回答数、正解数、不正解数、正答率、日別回答数、分野別正答率、目標達成率を表示してください。
```
