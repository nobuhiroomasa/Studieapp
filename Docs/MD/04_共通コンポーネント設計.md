# 勉強くん 共通コンポーネント設計

## 1. このファイルの目的

このドキュメントは、SwiftUIで実装する共通UI部品を定義する。

画面ごとに同じようなカード・ボタン・入力欄を毎回作ると、デザインがズレやすくなる。  
そのため、共通コンポーネントとして部品化し、全画面で統一して使う。

---

## 2. デザイン原則

| 項目 | 方針 |
|---|---|
| ベース | 白・薄いグレー背景 |
| メインカラー | 濃いブルー |
| カード | 白背景、角丸、軽い影 |
| ボタン | 大きめ、角丸、押しやすい |
| 正解 | グリーン系 |
| 不正解 | レッド系 |
| 文字 | 読みやすさ優先 |
| 操作 | 片手操作しやすい余白 |

# 2-1. 画面専用部品の扱い

以下のような部品は、初期版では共通コンポーネントとして必ず切り出す必要はない。  
各画面内の `private subview` として実装し、複数画面で再利用することが明確になった段階で `Components/` に移動する。

- `AppHeader`
- `TodayProgressCard`
- `ShortcutButton`
- `AppNavigationHeader`
- `QuizModeCard`
- `QuestionCountSelector`
- `CompleteHeroView`
- `ResultSummaryCard`
- `GoalProgressCard`
- `ReviewSummaryCard`
- `SearchBar`
- `FloatingAddButton`
- `StatCardGrid`
- `RecordChartTab`

`TertiaryButton` は初期版では作成しない。  
「ホームへ戻る」などの弱いアクションは `SecondaryButton` を利用する。

# 4. ボタン系コンポーネント

## 4-1. PrimaryButton

## 目的

最重要アクションに使う。

例：

- 問題を解く
- この条件で開始
- 次の問題へ
- 保存する

## Props

| 引数 | 型 | 内容 |
|---|---|---|
| `title` | `String` | ボタン文言 |
| `systemImage` | `String?` | アイコン |
| `isDisabled` | `Bool` | 無効状態 |
| `action` | `() -> Void` | タップ処理 |

## 実装イメージ

```swift
struct PrimaryButton: View {
    let title: String
    var systemImage: String? = nil
    var isDisabled: Bool = false
    let action: () -> Void

    var body: some View {
        Button(action: action) {
            HStack {
                if let systemImage {
                    Image(systemName: systemImage)
                }
                Text(title)
                    .fontWeight(.bold)
            }
            .frame(maxWidth: .infinity)
            .padding()
            .background(isDisabled ? Color.gray.opacity(0.3) : Color.appPrimary)
            .foregroundColor(.white)
            .clipShape(RoundedRectangle(cornerRadius: 16))
        }
        .disabled(isDisabled)
    }
}
```

# 5. カード系コンポーネント

## 5-1. AppCard

## 目的

全カードのベース。

## Props

| 引数 | 型 | 内容 |
|---|---|---|
| `content` | `ViewBuilder` | 中身 |

## 表示仕様

- 白背景
- 角丸16〜20
- 軽い影
- 内側余白16

## 実装イメージ

```swift
struct AppCard<Content: View>: View {
    let content: Content

    init(@ViewBuilder content: () -> Content) {
        self.content = content()
    }

    var body: some View {
        content
            .padding(16)
            .background(Color.appCardBackground)
            .clipShape(RoundedRectangle(cornerRadius: 18))
            .shadow(color: .black.opacity(0.06), radius: 8, x: 0, y: 4)
    }
}
```

# 12. 実装優先度

- `AppCard`
- `StatCard`
- `AnswerOptionButton`
- `CategoryChip`
- `QuizProgressBar`

## 優先度C

---

# 13. 注意点

1. 共通部品は画面デザイン画像と見た目がズレないようにする。
2. 画面ごとに似たボタンを個別実装しない。
3. 正解・不正解の色はアプリ全体で統一する。
4. フォームのバリデーション表示も統一する。
5. 最初は凝ったアニメーションより、押しやすさ・読みやすさを優先する。
